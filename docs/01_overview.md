# プロジェクト概要

## プロジェクト名
**Cupid** - 相思相愛マッチングLINE Bot

## 目的
- LINE Botを使った相思相愛マッチングアプリの開発
- AWSのインフラ学習（EC2管理、Nginx、SSL証明書、Linuxサーバー運用）
- Golangでのバックエンド開発

## コンセプト
自分の名前（ひらがな）・生年月日と、好きな人の名前（ひらがな）・生年月日を登録する。
相思相愛の場合のみ、両者に通知が届く。

## 技術スタック

### バックエンド
- **言語**: Go 1.21+
- **Webフレームワーク**: net/http（標準ライブラリ）
- **データベース**: SQLite 3
- **LINE SDK**: line-bot-sdk-go/v8

### インフラ (AWS)
- **EC2**: t4g.nano（ARM64 Graviton2、0.5GB RAM）
- **EBS**: gp3 10GB（OS + アプリ + データベース）
- **Nginx**: リバースプロキシ + SSL終端
- **Let's Encrypt**: SSL証明書（無料、自動更新）

### 外部サービス
- **LINE Messaging API**: ユーザーとのインターフェース
- **Route 53**: ドメイン管理（.click ドメイン、年$3）

## パフォーマンス要件
- メッセージ送信後、10秒以内に返信
- マッチング成立後、1分以内に通知
- 同時アクセス数は非常に低い（友達のみ使用）

## コスト要件
- 月額2,000円以内
- 予想コスト: 月620円程度
  - EC2 t4g.nano: $3.07
  - EBS gp3 10GB: $0.96
  - Route 53 ドメイン(.click): $0.25
  - Route 53 Hosted Zone: $0.50
- バックアップなし（コスト削減）

## スコープ外（後で考える）
- エッジケース処理
- 入力のバリデーション（生年月日フォーマットなど）
- 登録情報の変更・削除機能
- UIの細かい工夫や味付け

## 既知の制約・仕様

### 名前の表記
- **名前はひらがなで登録**する、よ
- 表記ゆれを減らすため（「高橋」vs「たかはし」→「たかはし」で統一）
- 例：「篠澤広」→「しのざわひろ」

### 同姓同名・同じ生年月日の場合
- 後から登録したユーザーの情報で**単純に上書き**される（意図的な仕様）
- 確認メッセージなし（シンプル優先）
- 友達同士で使う前提なら、確率は十分低い、よ

### 複数人への登録
- 1人のユーザーは1人しか好きな人を登録できない（上書きされる）
